@charset and @import

Concept & Syntax

@charset must be the very first rule in a CSS file and declares the encoding (now often handled by HTML/meta).

@import is used to load another CSS file (for example, a theme or fonts) into the current stylesheet. 

/* Very top of your main.css file */
@charset "UTF-8";

/* Import another stylesheet */
@import url("theme.css");

/* After that, your regular rules */
body {
  font-family: system-ui, sans-serif;
}
Explanation

In modern projects, @charset is rarely used because the HTML document and HTTP headers already define encoding.

@import is convenient but can slow loading because it adds extra HTTP requests. Using a single bundled CSS file is usually better.

@media ‚Äì responsive styles

@media applies styles only when certain conditions (screen width, orientation, etc.) are true. 

.card {
  padding: 1rem;
  background: #020617;
  color: #e5e7eb;
}

/* When viewport is at least 700px wide */
@media (min-width: 700px) {
  .card {
    display: flex;
    gap: 1rem;
  }
}

Resize the browser: below 700px, columns stack; above 700px, they go side-by-side.

This column represents content that will align next to Column B on larger screens.

On small screens, it stacks under Column A because the @media condition is not met.

@supports ‚Äì feature queries

@supports checks if the browser supports a given CSS property/value and applies styles conditionally. 

.card-supports {
  background: rgba(15,23,42,0.9);
}

/* Only if browser supports backdrop-filter */
@supports (backdrop-filter: blur(10px)) {
  .card-supports {
    backdrop-filter: blur(10px);
    background: rgba(15,23,42,0.5);
  }
}

@supports live card

If your browser supports backdrop-filter, this card will look like frosted glass.

@keyframes ‚Äì animations

@keyframes defines an animation sequence you can attach with the animation property. 

@keyframes pulse-demo {
  0%   { transform: scale(1);   background: #22c55e; }
  50%  { transform: scale(1.1); background: #16a34a; }
  100% { transform: scale(1);   background: #22c55e; }
}

.button-pulse {
  animation: pulse-demo 1.5s infinite;
}
@keyframes pulse-demo

The element grows and shrinks in a loop based on the 0%, 50%, 100% keyframes.

@font-face ‚Äì custom fonts

@font-face registers a local or external font file under a chosen font-family name. 

@font-face {
  font-family: "MyDemoFont";
  src: url("/fonts/MyDemoFont.woff2") format("woff2");
  font-weight: 400;
  font-style: normal;
}

.font-demo {
  font-family: "MyDemoFont", system-ui, sans-serif;
}

This text uses "MyDemoFont" if the file exists. If not, it falls back to your system UI font.

Replace /fonts/MyDemoFont.woff2 with your actual font path in a real project.

@page ‚Äì print pages

@page controls page size and margins when printing or exporting to PDF. 

@page {
  size: A4 portrait;
  margin: 20mm;
}

/* First printed page has extra top margin */
@page :first {
  margin-top: 40mm;
}

Use your browser‚Äôs ‚ÄúPrint‚Äù preview to see @page in action. Body content will be positioned with the specified margins.

@namespace ‚Äì XML/SVG namespaces (rare in app UI)

@namespace allows you to prefix selectors by XML namespace (useful in complex XML/SVG setups). 

@namespace svg url("http://www.w3.org/2000/svg");

/* Example: style only <svg:a> links */
svg|a {
  fill: red;
}

In typical HTML apps you rarely need @namespace.

@document ‚Äì URL-specific styles (non-standard)

@document was proposed to target specific URLs but is not widely supported and should generally be avoided. 

/* Example syntax, not for production use */
@document url("https://example.com/profile") {
  body {
    background: #0f172a;
  }
}

@layer ‚Äì cascade layers

@layer defines cascade layers (base, components, utilities) where later layers override earlier ones by default. 

@layer base {
  .layer-demo {
    color: #e5e7eb;
  }
}

@layer theme {
  .layer-demo {
    color: #f97316; /* wins because 'theme' layer comes later */
  }
}

The class in the theme layer overrides the color defined in the base layer.

@property ‚Äì register custom properties

@property allows you to register custom CSS properties (variables) with type, initial value, and inheritance. 

@property --glow-hue {
  syntax: "<number>";
  initial-value: 140;
  inherits: false;
}

.glow-chip {
  --glow-hue: 160;
  border-radius: 999px;
  padding: .5rem 1rem;
  border: 1px solid hsl(var(--glow-hue) 90% 50%);
  box-shadow: 0 0 10px hsl(var(--glow-hue) 90% 50% / .7);
}
Registered --glow-hue property

Because the property is registered, browsers can animate and interpolate it more reliably.

@container ‚Äì container queries

@container lets you style elements based on the size of a container instead of the whole viewport. 

.cq-article {
  container-type: inline-size;
}

.cq-grid {
  display: block;
}

@container (min-width: 550px) {
  .cq-grid {
    display: grid;
    grid-template-columns: 2fr 1fr;
    gap: .75rem;
  }
}

Combining @media with normal rules

You can write regular rules, then add @media blocks that override them for specific ranges. 

.nesting-demo {
  color: #e5e7eb;
}

/* Larger screens: change color */
@media (min-width: 600px) {
  .nesting-demo {
    color: #a5b4fc;
  }
}

On small screens this text is light gray; on larger screens it changes to light indigo.

@scope ‚Äì experimental scoping

@scope is new and not fully supported; it limits which elements certain selectors apply to. 

@scope (.scope-root) {
  .scoped-title {
    color: #f97316;
  }
}

Scoped title (inside .scope-root)

Unscoped title (outside .scope-root) ‚Äì may not be affected depending on browser support.

Treat @scope as experimental for now and verify browser support before using in production.
