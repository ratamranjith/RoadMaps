3.1: When to Use Manual Testing

Manual testing means a human tester manually executes test cases without automation tools. It's still essential in modern software development.

Best Use Cases for Manual Testing

  • ‚úÖ Exploratory Testing: Testing without predefined test cases, discovering issues dynamically
  • ‚úÖ Usability Testing: Evaluating user experience and interface design
  • ‚úÖ Ad-hoc Testing: Testing based on tester's intuition and experience
  • ‚úÖ Complex Scenarios: Test cases that are difficult to automate
  • ‚úÖ New Features: Testing new features before automation is created
  • ‚úÖ Visual Testing: Checking colors, fonts, alignment, responsive design
  • ‚úÖ User Flow Testing: Testing complete user journeys and workflows

When NOT to Use Manual Testing

Scenario Why Automation is Better
Repetitive test execution Automation saves time and reduces human error
Regression testing Same tests run hundreds of times
Performance/Load testing Need to simulate thousands of concurrent users
Large test suites Manual execution would take too long
Cross-browser testing Automation handles multiple browsers automatically
The Truth About Manual Testing: It's not going away. The best teams use BOTH manual and automation testing together.

3.2: Exploratory Testing

Exploratory testing is simultaneous learning, test design, and test execution. The tester explores the application without predefined test cases.

Exploratory Testing Process

  1. Learn the Feature: Understand what the feature is supposed to do
  2. Ask Questions: What could go wrong? What are edge cases?
  3. Test Freely: Try different combinations and flows
  4. Observe Behavior: Notice unexpected behavior
  5. Document Issues: Report bugs found
  6. Repeat with Focus: Dive deeper into risky areas
Example - Exploratory Testing for E-Commerce Checkout:
  • ‚Ä¢ What if I add 1000 items to cart?
  • ‚Ä¢ What if I use special characters in address?
  • ‚Ä¢ What if I open checkout in two tabs simultaneously?
  • ‚Ä¢ What if I refresh page during payment?
  • ‚Ä¢ What if I use very old browser?

3.3: Real-World E-Commerce Testing

Here's how a professional tester approaches testing an e-commerce checkout flow.

Happy Path (Everything Works)

Test: Successful purchase 1. Browse products and add item to cart 2. Proceed to checkout 3. Enter shipping address 4. Select shipping method (Standard/Express) 5. Enter payment information 6. Review order summary 7. Click "Place Order" Expected: Order confirmed, email sent, redirected to confirmation page

Edge Cases & Error Scenarios

  • ‚ùå Empty Cart: Try to checkout with empty cart ‚Üí Should show error
  • ‚ùå Out of Stock: Item goes out of stock after adding ‚Üí Warn user
  • ‚ùå Invalid Address: Enter incomplete/invalid address ‚Üí Show validation error
  • ‚ùå Expired Card: Use expired credit card ‚Üí Payment declined
  • ‚ùå Session Timeout: Wait 30 minutes mid-checkout ‚Üí Redirect to login
  • ‚ùå Browser Back Button: Click back during payment ‚Üí Handle gracefully
  • ‚ùå Network Interruption: Lose internet during checkout ‚Üí Retry mechanism
  • ‚ùå Duplicate Order: Double-click "Place Order" ‚Üí Only charge once
  • ‚ùå Invalid Promo Code: Enter wrong coupon ‚Üí Show error message
  • ‚ùå Mobile Issues: Test on different screen sizes ‚Üí Responsive layout

3.4: Professional Bug Reporting

A good bug report is clear, detailed, and actionable. It should be reproducible by anyone.

Bug Report Template

Title: [CRITICAL] Payment button unresponsive on Firefox Environment: - Browser: Firefox 122.0 - OS: Windows 11 - Screen Resolution: 1920x1080 - App Version: 2.5.1 Severity: CRITICAL (Feature broken) Priority: P1 (Urgent) Description: The "Place Order" button on the checkout page does not respond to clicks when using Firefox browser. The button appears to be clickable but no action is triggered. Steps to Reproduce: 1. Open https://example.com/checkout in Firefox 2. Fill in all required fields (shipping, payment) 3. Click the "Place Order" button 4. Observe: Button remains in same state, no loading indicator, no navigation Expected Behavior: - Button should show loading state (spinner) - Page should redirect to order confirmation - Success message should appear Actual Behavior: - Button does nothing when clicked - No loading indicator - No error message shown - Page remains on checkout Screenshots: [Attached: checkout_button_firefox.png] [Attached: console_errors.png] Console Errors: TypeError: Cannot read properties of null (reading 'addEventListener') at checkout.js:42 Workaround: - Use Chrome or Safari (works fine) - Not a workaround for Firefox users Additional Notes: - Works fine in Chrome 120 - Works fine in Safari 17 - Only issue is Firefox - Affects 15-20% of users based on analytics

Bug Report Best Practices

  • ‚úì Clear Title: Starts with severity level [CRITICAL], [MAJOR], etc.
  • ‚úì Environment: Browser, OS, version, screen size
  • ‚úì Reproducible Steps: 5-10 clear steps anyone can follow
  • ‚úì Expected vs Actual: What should happen vs what does happen
  • ‚úì Screenshots/Videos: Visual evidence of the issue
  • ‚úì Console Errors: Any JavaScript errors or warnings
  • ‚úì Frequency: Does it happen every time or intermittently?
  • ‚úì Workaround: Temporary solution if one exists
  • ‚úì Impact: How many users are affected?
‚ö†Ô∏è What NOT to Do:
  • ‚Ä¢ Don't write vague titles like "Button doesn't work"
  • ‚Ä¢ Don't forget to include steps to reproduce
  • ‚Ä¢ Don't report bugs on untested hardware/browsers
  • ‚Ä¢ Don't blame developers in the bug report
  • ‚Ä¢ Don't report the same bug multiple times

3.5: Manual Testing Checklist

Use this checklist for any feature you're testing manually.

GENERAL ‚òê Feature works as specified in requirements ‚òê Feature works on all required browsers ‚òê Feature works on mobile devices ‚òê No console errors or warnings ‚òê No broken links FUNCTIONALITY ‚òê All buttons work correctly ‚òê All forms submit successfully ‚òê All validations work (required fields, formats) ‚òê Error messages are clear and helpful ‚òê Success messages appear after actions USER EXPERIENCE ‚òê Loading indicators appear for long operations ‚òê Navigation is intuitive ‚òê Form fields are properly labeled ‚òê Button text is clear (not just "Click here") ‚òê Confirmation messages for destructive actions ACCESSIBILITY ‚òê Can navigate using keyboard (Tab key) ‚òê Color contrast is sufficient ‚òê Font size is readable ‚òê Images have alt text PERFORMANCE ‚òê Pages load quickly (< 3 seconds) ‚òê No lag when interacting with elements ‚òê Images are properly optimized SECURITY ‚òê Sensitive data is not visible in URLs ‚òê Sessions timeout appropriately ‚òê Can't access restricted pages without permission
