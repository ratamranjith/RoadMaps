6.1: HTTP Methods & Status Codes

APIs communicate using HTTP methods. Understanding them is crucial for effective testing.

HTTP Methods

Method Purpose Idempotent Example
GET Retrieve data Yes /users/1
POST Create resource No /users (create new)
PUT Replace resource Yes /users/1 (full update)
PATCH Partial update Yes /users/1 (partial update)
DELETE Remove resource Yes /users/1 (delete)

HTTP Status Codes

2xx - Success: 200 OK, 201 Created, 204 No Content
3xx - Redirection: 301 Moved Permanently, 302 Found
4xx - Client Error: 400 Bad Request, 401 Unauthorized, 403 Forbidden, 404 Not Found
5xx - Server Error: 500 Internal Error, 502 Bad Gateway, 503 Service Unavailable

6.2: Postman - API Testing Tool

Postman is the most popular tool for API testing and documentation.

Basic API Test in Postman

STEP 1: Create Request - Method: GET - URL: https://jsonplaceholder.typicode.com/users/1 - Click Send STEP 2: Add Headers (if needed) - Key: Content-Type - Value: application/json - Key: Authorization - Value: Bearer YOUR_TOKEN STEP 3: Add Test (JavaScript) pm.test("Status code is 200", () => { pm.expect(pm.response.code).to.equal(200); }); pm.test("Response has user ID", () => { let jsonData = pm.response.json(); pm.expect(jsonData.id).to.equal(1); }); pm.test("Response time is less than 500ms", () => { pm.expect(pm.response.responseTime).to.be.below(500); });

6.3: API Testing Checklist

Request Level

  • ‚úì Correct HTTP method used
  • ‚úì Valid endpoint URL
  • ‚úì Required headers present
  • ‚úì Request body (if POST/PUT) is valid
  • ‚úì Authentication token included
  • ‚úì Query parameters correct

Response Level

  • ‚úì Status code is correct (200, 201, 400, etc.)
  • ‚úì Response headers present (Content-Type, etc.)
  • ‚úì Response body matches expected schema
  • ‚úì Required fields are present
  • ‚úì Data types are correct
  • ‚úì No sensitive data exposed
  • ‚úì Response time acceptable (< 1 second)

Error Scenarios

  • ‚úì Invalid input returns 400
  • ‚úì Missing auth returns 401
  • ‚úì Insufficient permissions returns 403
  • ‚úì Resource not found returns 404
  • ‚úì Server error returns 5xx
  • ‚úì Error message is clear

6.4: Real API Testing Examples

// GET request - Retrieve user GET /api/v1/users/123 Headers: Authorization: Bearer token123 Accept: application/json Response 200: { "id": 123, "name": "John Doe", "email": "john@example.com", "created_at": "2024-01-15" }
// POST request - Create user POST /api/v1/users Headers: Content-Type: application/json Authorization: Bearer token123 Body: { "name": "Jane Smith", "email": "jane@example.com", "password": "SecurePass123" } Response 201: { "id": 124, "name": "Jane Smith", "email": "jane@example.com" }
// DELETE request - Remove user DELETE /api/v1/users/123 Headers: Authorization: Bearer token123 Response 204: (No content)
